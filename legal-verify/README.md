# legal-verify — 法条真实性与有效性核验技能

> **版本**：1.2.0 | **作者**：浮生模样（微信公众号 fsmy-2020）| **许可**：MIT

## 简介

法条核验技能，用于核实法律条文的真实性、有效性与法律适用正确性。当用户粘贴法律条文、引用法规、询问某条法律是否正确，或上传含有法律引用的文件时自动触发。从官方权威来源（国家法律法规数据库、全国人大官网等）获取原文进行比对，确认内容是否准确、法律是否现行有效，并检测是否存在司法解释改变理解方式的情况。

### 核心能力

- **单条核验** — 粘贴一条法条，逐字比对官方原文
- **批量核验** — 上传 docx/pdf/txt/md 文件，自动提取所有法律引用并逐条核验
- **版本追踪** — 自动识别 2020-2026 年重大修法，对条款编号变动发出预警
- **司法解释检测** — 双轨搜索（法条级+关键词级），发现司法解释改变理解方式的情况
- **用户适用意图** — 了解用户如何理解/适用法条，精准判定解释性差异
- **置信度评估** — 高/中/低三级置信度，低置信度不输出确定性结论
- **专业报告** — 支持 Markdown 快速报告和 DOCX 排版报告双输出

### 文本差异分级

| 级别 | 含义 | 标记 |
|------|------|------|
| ID | 完全一致 | 🟢 |
| MN | 轻微差异（标点/格式） | 🟡 |
| SD | 实质差异（关键词/数字） | 🟠 |
| CD | 严重差异（逻辑词互换） | 🔴 |
| UV | 无法核实 | ⚪ |

### 解释性差异分级（v1.1.0 新增，v1.2.0 增强）

| 级别 | 含义 | 标记 |
|------|------|------|
| IC-CD | 司法解释否定了用户理解 | 🔴 |
| IC-SD | 司法解释改变了适用条件 | 🟠 |
| IC-MN | 司法解释补充了未明确事项 | 🟡 |
| IC-Info | 配套规定细化了适用标准 | 🟢 |
| IC-Pending | 发现但未判定（用户未提供适用意图） | ⚪ |

> **核心原则**：文本完全一致 ≠ 法律适用正确。即便法条文本一字不差、法律现行有效，相关司法解释可能已改变该条款的理解方式。

## v1.2.0 更新说明

v1.2.0 解决了一个致命缺陷：v1.1.0 虽然增加了司法解释搜索功能，但在实际测试中未能发现已知案例（劳动合同法第14条"订立"vs"延长"的解释性差异）。

**根因分析**：
1. 不知道用户的法律适用意图 → 无法判断司法解释是否与用户理解冲突
2. 搜索策略只有法条级 → 搜"第14条司法解释"返回笼统结果，无法精准命中
3. 司法解释检测是步骤4的子步骤 → 容易被AI跳过

**v1.2.0 三大修复**：
1. **新增"用户适用意图"输入** — 询问用户基于该条款的法律论点，才能判断司法解释是否冲突
2. **新增"关键词级司法解释搜索"** — 用"订立 延长 司法解释"替代"第14条司法解释"，精准命中
3. **司法解释检测升级为独立步骤** — 从步骤4.3→步骤五，不可跳过

## 使用方法

### 触发方式

直接粘贴法条或提出核验需求即可触发，例如：

- "帮我核验一下《劳动合同法》第47条"
- "这条法律对不对：经济补偿按劳动者在本单位工作的年限……"
- "核验法条：根据《公司法》第16条……我认为担保只需董事会决议即可"
- 上传包含法律引用的文件

> 💡 **v1.2.0 提示**：如果能说明你基于该条款的法律论点或适用方式（如"我认为订立不包含延长"），核验结果会更精准——技能可以判断司法解释是否与你的理解冲突。

### 批量核验

上传文件后，技能自动调用 `scripts/parse_citations.py` 提取法律引用，逐条核验后汇总输出。

### 生成 DOCX 报告

核验完成后，可将结构化 JSON 结果通过脚本生成专业排版报告：

```bash
python scripts/generate_report.py result.json --output 核验报告.docx --title "法条核验报告"
```

### 单独使用引用提取脚本

```bash
python scripts/parse_citations.py 文件路径.docx --output citations.json
```

支持格式：.docx / .pdf / .txt / .md

## 工作流

```
步骤一：解析法条信息 + 用户适用意图 → CP1（确认法律全称）
步骤二：搜索官方原文 → CP2（确认最新版本）
步骤三：版本敏感法条速查（自动预警）
步骤四：执行核查与比对（文本+有效性+置信度）
步骤五：司法解释与适用性审查（双轨搜索+关键词级搜索）→ CP3（双轨搜索完成）→ CP4（适用意图已确认）
步骤六：输出核验报告 → CP5（12项自检）→ CP6（置信度匹配）
```
