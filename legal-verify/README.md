# legal-verify — 法条校验技能

> **版本**：2.0.0 | **作者**：浮生模样（微信公众号 fsmy-2020）| **许可**：MIT

## 简介

法条校验技能，对文本中援引的法律条文执行**对抗性验证**（adversarial verification）——主动寻找引用错误，而非被动接受。覆盖条文原文比对、时效性核查、司法解释检测、管辖区适配、援引格式规范化，以及按严重程度分级的错误报告。当用户粘贴法律条文、引用法规、询问某条法律是否正确，或上传含有法律引用的文件时自动触发。

### 核心能力

- **对抗性验证** — 主动寻找引用错误，非被动接受
- **引用可信度标注** — 实时核实标注 `[来源: XXX]`，模型推断标注 `[待核实]`
- **单条/批量核验** — 粘贴一条法条或上传文件，逐字比对官方原文
- **时效链追踪** — `[原始法律] → [历次修订] → [当前有效版本] → [是否整合入新法]`
- **6类错误检测** — 原文曲解/条款错位/版本错误/格式不规范/引用冗余/管辖区错配
- **司法解释检测** — 多轨搜索（法条级+关键词级+配套规定+指导案例）
- **用户适用意图** — 了解用户如何理解/适用法条，精准判定解释性差异
- **管辖区适配** — 检查法域错配、地方性法规冲突、准据法缺失
- **格式规范化** — 检查引用格式是否符合中国法律文书标准
- **校验深度选项** — 快速/标准/深度三档，适配不同审查场景
- **可分发性评估** — 直接分发/修正后分发/需全面复核
- **置信度评估** — 高/中/低三级置信度，低置信度不输出确定性结论
- **专业报告** — 支持 Markdown 快速报告和 DOCX 排版报告双输出

### 文本差异分级

| 级别 | 含义 | 标记 |
|------|------|------|
| ID | 完全一致 | 🟢 |
| MN | 轻微差异（标点/格式） | 🟡 |
| SD | 实质差异（关键词/数字） | 🟠 |
| CD | 严重差异（逻辑词互换） | 🔴 |
| UV | 无法核实 | ⚪ |

### 解释性差异分级

| 级别 | 含义 | 标记 |
|------|------|------|
| IC-CD | 司法解释否定了用户理解 | 🔴 |
| IC-SD | 司法解释改变了适用条件 | 🟠 |
| IC-MN | 司法解释补充了未明确事项 | 🟡 |
| IC-Info | 配套规定细化了适用标准 | 🟢 |
| IC-Pending | 发现但未判定（用户未提供适用意图） | ⚪ |

### 错误类型分类（v2.0.0 新增）

| 类型 | 说明 | 严重程度 |
|------|------|---------|
| 原文曲解 | 条文含义被扩大/缩小/反转 | 🔴 严重 |
| 条款错位 | 引用内容属于相邻条款 | 🔴 严重 |
| 版本错误 | 引用旧版条文 | 🟠 重要 |
| 格式不规范 | 编号格式不符合规范 | 🟡 轻微 |
| 引用冗余 | 引用条款与论证无实质关联 | 🟡 轻微 |
| 管辖区错配 | 引用法律与适用管辖区不符 | 🔴 严重 |

> **核心原则**：文本完全一致 ≠ 法律适用正确。即便法条文本一字不差、法律现行有效，相关司法解释可能已改变该条款的理解方式。

## v2.0.0 更新说明

v2.0.0 在 v1.2.0 基础上全面升级，从"核验"定位升级为"对抗性验证"定位，新增4大维度：

**1. 对抗性验证 + 引用可信度标注**
- 定位从被动核验升级为主动寻找错误的对抗性验证
- 新增引用可信度标注体系：`[来源: XXX]` / `[待核实]` / 无数据库时头部声明

**2. 管辖区适配检查（步骤五）**
- 检查法域错配、地方性法规冲突、准据法缺失
- 管辖区未明确时必须显式标注推断依据，不得静默假设

**3. 格式规范化建议（步骤六）**
- 检查引用格式是否符合中国法律文书标准
- 提供规范化修正建议

**4. 增强报告体系**
- 新增6类错误类型分类表
- 新增可分发性评估（直接分发/修正后分发/需全面复核）
- 新增时效链追踪格式
- 新增校验局限性说明
- 检查清单从12项扩展到14项（C13管辖区、C14格式）
- 工作流从6步扩展到7步
- 检查点从6个扩展到7个

**保留 v1.2.0 全部核心能力**：用户适用意图、关键词级司法解释搜索、多轨搜索策略、解释性差异(IC)维度、IC-Pending、引用提取脚本、报告生成脚本、完整JSON Schema。

## 使用方法

### 触发方式

直接粘贴法条或提出校验需求即可触发，例如：

- "帮我核实一下《劳动合同法》第47条"
- "这条法律对不对：经济补偿按劳动者在本单位工作的年限……"
- "校验法条：根据《公司法》第16条……我认为担保只需董事会决议即可"
- "检查这份合同中的法条引用" + 上传文件
- "核实法条" / "校验法律依据" / "法规是否有效"

### 校验深度

| 深度 | 触发方式 | 覆盖范围 |
|-----|---------|---------|
| 快速 | `--depth 快速` | 时效性 + 明显错误 |
| 标准 | 默认 | 全流程七步 |
| 深度 | `--depth 深度` 或合同/诉状/法律意见书自动推断 | 全流程 + 四轨司法解释搜索 |

### 批量核验

上传文件后，技能自动调用 `scripts/parse_citations.py` 提取法律引用，逐条校验后汇总输出。

### 生成 DOCX 报告

校验完成后，可将结构化 JSON 结果通过脚本生成专业排版报告：

```bash
python scripts/generate_report.py result.json --output 校验报告.docx --title "法条校验报告"
```

### 单独使用引用提取脚本

```bash
python scripts/parse_citations.py 文件路径.docx --output citations.json
```

支持格式：.docx / .pdf / .txt / .md

## 工作流

```
步骤一：提取与结构化引用清单 + 用户适用意图 + 初始风险标注 → CP1（确认法律全称）
步骤二：时效性验证（时效链追踪+搜索官方原文+版本预警） → CP2（确认最新版本）
步骤三：原文比对（6类错误检测+文本差异分级+有效性核查） → 置信度评估
步骤四：司法解释与适用性审查（多轨搜索+关键词级搜索） → CP3（搜索完成）→ CP4（意图确认）
步骤五：管辖区适配检查 → CP5（管辖区检查完成）
步骤六：格式规范化建议
步骤七：输出校验报告 → CP6（14项自检）→ CP7（置信度匹配）
```

## 边界与护栏

- **不提供法律意见**：校验结论仅针对引用准确性，不判断法律策略妥当性
- **不直接修改源文件**：仅提供修正建议，实际修改由律师决定并执行
- **司法解释谨慎处理**：基于模型知识核实的司法解释一律标注 `[待核实]`
- **争议法律领域**：标注"存在解释争议"，不作倾向性结论
- **保密提示**：提醒使用外部数据库核实时注意保密义务
