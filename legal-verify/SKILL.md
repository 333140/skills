# 法条真实性与有效性核验技能

## 核验模式识别

- **单条核验**：用户直接粘贴或引用法律条文 → 执行步骤一至步骤五
- **文件批量核验**：用户上传文件（.docx/.pdf/.txt/.md 等）→ 先执行【文件解析】，再对每条引用重复步骤二至步骤四，最后执行步骤五汇总输出

---

## 步骤一：解析法条信息

### 1.1 从用户文本中提取

| 字段 | 说明 | 必填 |
|------|------|------|
| **法律名称** | 完整名称，如《中华人民共和国劳动合同法》 | ✅ |
| **条款编号** | 第X条/第X款/第X项 | ✅ |
| **颁布机关** | 用于判断搜索来源（全国人大/国务院/部委/地方） | ❌（可推断） |
| **版本年份** | 如用户注明 | ❌ |
| **条文内容** | 用户提供的具体文字 | ✅ |

> **检查点 CP1**：法律名称必须使用全称（含"中华人民共和国"前缀）。若用户使用简称（如"劳动法"），须确认是指《劳动法》还是《劳动合同法》，避免混淆。

### 1.2 文件批量模式 — 引用提取

对上传的文件，使用 `scripts/parse_citations.py` 提取法律引用：

```bash
python scripts/parse_citations.py <文件路径> --output citations.json
```

脚本支持 8 种引用模式的正则提取：
1. 依据/根据《XX法》第X条/款/项
2. 《XX法》第X条第X款第X项
3. 《XX法》第X款
4. 《XX法》第X项
5. 《XX法》规定/相关规定
6. 《XX法》第X条至第X条（范围引用）
7. XX法第X条（无书名号）
8. 《XX法》法释〔年份〕X号（司法解释号引用）

脚本自动按法律名称分组去重，输出标准化 JSON。

> 若脚本不可用或用户仅粘贴文本，则手动扫描以下模式：
> - "依据/根据《XX法》第X条/款/项"
> - "依据/根据《XX法》规定"
> - "《XX法》第X条"
> - 按法律名称分组，避免重复搜索

## 步骤二：搜索官方原文

### 2.1 搜索策略

**必须并行发起至少 2 个搜索**（使用并行 tool call），优先从官方来源获取。

搜索查询模板（详见 `references/law-database-urls.md`）：

```
// 并行搜索组合1：法律全文
web_search("site:flk.npc.gov.cn {法律全称}")
web_search("site:npc.gov.cn {法律全称} 全文")
web_search("{法律全称} 现行有效 最新版本")
```

```
// 并行搜索组合2：具体条款
web_search("{法律全称} 第{X}条 全文 官方")
web_search("site:flk.npc.gov.cn {法律全称} 第{X}条")
```

```
// 并行搜索组合3：废止/修订状态
web_search("{法律全称} 修订 废止 最新版本")
web_search("{法律全称} 现行有效")
```

### 2.2 按法律层级确定来源

| 层级 | 示例 | 权威来源（按优先级） |
|------|------|---------------------|
| 宪法 / 法律 | 民法典、刑法 | flk.npc.gov.cn → npc.gov.cn |
| 行政法规 | 国务院条例 | gov.cn（政策法规库）→ moj.gov.cn |
| 部门规章 | 各部委规定 | 对应部委官网 → gov.cn |
| 地方性法规 | 省级人大条例 | 省级人大官网 → 通用搜索 |
| 司法解释 | 最高院规定 | court.gov.cn → 通用搜索 |

> 详见 `references/law-database-urls.md` 中各来源的具体 URL 和搜索模板。

### 2.3 港澳台及境外法律

| 地区 | 官方来源 | 搜索模板 |
|------|---------|---------|
| 香港 | elegislation.gov.hk | `site:elegislation.gov.hk {law name}` |
| 澳门 | bo.io.gov.mo | `site:bo.io.gov.mo {法律名称}` |
| 台湾 | law.moj.gov.tw | `site:law.moj.gov.tw {法律名称}` |
| 美国联邦 | uscode.house.gov / law.cornell.edu | `site:law.cornell.edu {act name}` |
| 欧盟 | eur-lex.europa.eu | `site:eur-lex.europa.eu {regulation number}` |
| 其他 | 通用搜索 | `web_search("{country} {law name} official full text site:gov")` |

### 2.4 搜索结果获取

从搜索结果中识别官方来源页面 URL，使用 `web_fetch` 获取法律全文。

若官方网站无法访问或未找到，按以下顺序降级：
1. 第二优先官方来源
2. 辅助参考来源（北大法宝、法信）— 仅作交叉验证
3. 通用搜索结果 — 标记为低置信度

> **检查点 CP2**：必须确认获取的是**最新版本**的官方原文。参考 `references/frequently-revised-laws.md` 中的修法时间线，确认版本对应。

## 步骤三：版本敏感法条速查

> ⚠️ 本步骤与步骤二可并行执行。对以下法律，**必须**额外执行版本敏感检查。

### 3.1 自动预警触发

当核验的法律在 `references/frequently-revised-laws.md` 中被标记为🔴极高敏感度或🟡中等敏感度时，自动触发以下额外检查：

**🔴 极高敏感度法律**（条款重排，旧号几乎全部失效）：
- 《公司法》（2023年修订，2024.7.1施行）— 条款从218条→266条
- 《行政复议法》（2023年修订，2024.1.1施行）— 条款从43条→86条
- 民法典相关旧法引用（婚姻法、继承法、合同法、物权法等→民法典）

**🟡 中等敏感度法律**（部分条款变动）：
- 《劳动合同法》（2012年修正，劳务派遣部分有调整）
- 《行政处罚法》（2021年修订）
- 《反垄断法》（2022年修正）
- 《安全生产法》（2021年修正）
- 《专利法》（2020年修正）
- 《著作权法》（2020年修正）

### 3.2 版本检查操作

```
web_search("{法律名称} 修订历史 修订时间线")
web_search("{法律名称} 现行有效 最新版本 {当前年份}")
```

确认：
1. 用户引用的版本是否为现行有效版本
2. 条款编号在修订后是否发生了变动
3. 若条款已移位，标注新旧条款编号对照

> 详见 `references/frequently-revised-laws.md` 中的条款编号变动对照表。

## 步骤四：执行核查与比对

### 4.1 内容真实性比对

获取官方原文后，按 `references/comparison-rules.md` 中的规则进行系统性比对。

**比对维度与重点**：

| 比对维度 | 重点核查项 | 严重程度 |
|---------|-----------|---------|
| 逻辑连词 | "且"↔"或"、"并"↔"或" | 🔴 严重 |
| 模态词 | "应当"↔"可以"、"有权"↔"应当" | 🔴 严重 |
| 否定词 | "不得"↔"不应"、"未"↔"不" | 🟡 实质 |
| 数字期限 | 日数、年限、比例、金额、人数 | 🔴 严重 |
| 主体名称 | "法人"↔"法定代表人"、"公司"↔"企业" | 🔴/🟡 |
| 但书/除外 | 遗漏但书改变了适用范围 | 🔴 严重 |
| 条款对应 | 条款编号与内容是否对应 | 🔴 严重 |
| 款项层级 | 引用精度（条/款/项） | 🟡 实质 |
| 标点空格 | 逗号/句号/分号互换 | 🟢 轻微 |

**比对结果分级**：

| 级别 | 代号 | 含义 | 颜色标记 |
|------|------|------|---------|
| 完全一致 | ID | 逐字一致 | 🟢 |
| 轻微差异 | MN | 标点/格式差异，意思不变 | 🟡 |
| 实质差异 | SD | 关键词/数字/主体差异 | 🟠 |
| 严重差异 | CD | 逻辑词/模态词互换，意思相反 | 🔴 |
| 无法核实 | UV | 未找到官方原文 | ⚪ |

> 详见 `references/comparison-rules.md` 中的完整敏感词表和分级判定流程。

### 4.2 法律有效性核查

确认以下五项（缺一不可）：

- [ ] 该法律是否仍然有效（未被废止）
- [ ] 是否有更新版本（用户引用的可能是旧版）
- [ ] 用户引用的具体条款在最新版中是否变动、删除或移位
- [ ] 施行日期（确认已生效）
- [ ] 若已废止：找到替代的新法律及对应条款

**废止法律→替代法律映射**（详见 `references/frequently-revised-laws.md`）：
- 《婚姻法》《继承法》《合同法》《物权法》《侵权责任法》《担保法》《收养法》《民法通则》《民法总则》→ 《民法典》各编
- 《执业医师法》→ 《医师法》
- 《外资企业法》《中外合资经营企业法》《中外合作经营企业法》→ 《外商投资法》

### 4.3 置信度评估

根据核验过程中的来源可靠性和完整性，评估结果置信度：

| 等级 | 代号 | 条件 |
|------|------|------|
| **高** | H | ≥2个官方来源 + 来源一致 + 确认现行有效 + 确认最新版本 + 逐字比对完成 + 但书已检查 |
| **中** | M | 1个官方+1个辅助来源一致 / 2个辅助来源一致 / 无法确认最新版本 |
| **低** | L | 仅1个非官方来源 / 来源不一致 / 法律名称不确定 |

> 详见 `references/verification-checklist.md` 中的完整评估标准和因子表。

## 步骤五：输出核验报告

### 5.1 核验自检

输出报告前，**必须**对照 `references/verification-checklist.md` 完成以下 8 项检查：

| 序号 | 检查项 | 常见遗漏 |
|------|--------|---------|
| C1 | 法律全称确认 | 用简称代替全称 |
| C2 | 官方原文获取 | 仅从非官方来源获取 |
| C3 | 来源数量≥2 | 仅搜了1次就下结论 |
| C4 | 废止状态确认 | 忘记查废止状态 |
| C5 | 最新版本确认 | 获取了旧版而非最新版 |
| C6 | 条款编号验证 | 引用了旧版条款号 |
| C7 | 内容逐字比对 | 仅做了大意比对 |
| C8 | 例外/但书检查 | 遗漏但书改变适用范围 |

**通过标准**：
- ✅ 8项全部确认 → 可输出确定性结论
- ⚠️ 6-7项确认 → 输出结论但标注未确认项
- ❌ 5项以下 → 不可输出核验结论，仅列出已获取信息

### 5.3 DOCX 报告生成

将整理好的 JSON 保存到文件，然后调用脚本生成专业排版报告：

```bash
python scripts/generate_report.py result.json --output 核验报告.docx --title "法条核验报告"
```

报告功能：
- 封面（法律名称 + 核验日期 + 置信度等级）
- 核验对象信息表
- 核验来源表（来源名称 + URL + 访问状态）
- 官方原文引用块
- 差异比对表（用户版 vs 官方版，差异级别颜色标记）
- 有效性状态表（🟢/🟡/🔴 状态指示）
- 置信度评估说明（因子详情表）
- 核验结论
- 免责声明
- 批量模式额外：文件概况 + 核验结果概览表 + 总结建议分类

## 工作流检查点汇总

## 特殊情况处理

### 无法找到官方原文时

> ⚪ **未能核实**：在官方数据库中未能找到《XXX》第X条的官方原文。可能原因：(1) 该法条不存在于现行法律体系；(2) 属于内部规定或地方性规范性文件；(3) 法律名称有误。建议通过 [flk.npc.gov.cn](https://flk.npc.gov.cn) 进行人工查询，或咨询专业律师。

置信度标记为 L，不可输出确定性结论。

### 版本敏感提示

许多法律经历多次修订（如《著作权法》《公司法》《民事诉讼法》等），用户可能引用旧版条文。即便内容本身曾经存在，也须明确指出是否为**当前有效版本**。

参考 `references/frequently-revised-laws.md` 中的修法时间线，对🔴极高敏感度和🟡中等敏感度法律自动触发额外版本检查。

### 条款编号变动

修订后条款编号可能移位（如《公司法》2023年修订后多处条款编号变动），须明确告知用户当前版本中的正确条款位置。

参考 `references/frequently-revised-laws.md` 中的条款编号变动对照表。

### 多版本条文共存

部分法律存在新旧版本过渡期（如《公司法》2023年修订版2024年7月1日施行，旧版在此日期前仍有效）。须根据用户引用的具体时间场景判断适用版本。
