# 法条真实性与有效性核验技能

> **v1.2.0 核心变更**：新增"用户适用意图"输入维度和"关键词级司法解释搜索"策略，解决"法条文本正确但法律适用错误"的检测盲区。

## 核验模式识别

- **单条核验**：用户直接粘贴或引用法律条文 → 执行步骤一至步骤六
- **文件批量核验**：用户上传文件（.docx/.pdf/.txt/.md 等）→ 先执行【文件解析】，再对每条引用重复步骤二至步骤五，最后执行步骤六汇总输出

---

## 步骤一：解析法条信息

### 1.1 从用户文本中提取

| 字段 | 说明 | 必填 |
|------|------|------|
| **法律名称** | 完整名称，如《中华人民共和国劳动合同法》 | ✅ |
| **条款编号** | 第X条/第X款/第X项 | ✅ |
| **颁布机关** | 用于判断搜索来源（全国人大/国务院/部委/地方） | ❌（可推断） |
| **版本年份** | 如用户注明 | ❌ |
| **条文内容** | 用户提供的具体文字 | ✅ |
| **用户适用意图** | 用户基于该条款的法律论点、适用方式或理解 | ⚠️ 强烈建议 |

> **检查点 CP1**：法律名称必须使用全称（含"中华人民共和国"前缀）。若用户使用简称（如"劳动法"），须确认是指《劳动法》还是《劳动合同法》，避免混淆。

### 1.2 用户适用意图（v1.2.0 核心新增）

> ⚠️ **这是v1.2.0解决司法解释检测盲区的关键输入。**

**为什么需要用户的适用意图？**

法条核验存在两种截然不同的错误类型：
1. **文本错误**：法条文本与官方原文不一致 → 不需要用户意图即可检测
2. **适用错误**：法条文本完全正确，但司法解释已改变/明确了该条款的理解方式 → **必须知道用户怎么理解/适用该条款**，才能判断是否存在解释性差异

**如何获取用户适用意图？**

- **用户主动提供**：如"我认为订立不包含延长""依据该条主张XX权利"
- **从上下文推断**：用户提到"根据第X条，所以YY"→ "YY"就是适用意图
- **若用户未提供**：**必须主动询问**——"请问您基于该条款的法律论点或适用方式是什么？这有助于检测是否存在司法解释改变了该条款的理解方式。"
- **批量模式**：若文件中包含法律论述，从论述中提取适用意图；若仅列出法条引用无论述，标记为"未提供"

**用户适用意图示例**：

| 法条 | 用户的适用意图 | 可能的解释性差异 |
|------|--------------|----------------|
| 劳动合同法第14条第2款第3项 | "订立"指重新签订合同，延长协议不计入"连续订立"次数 | 法释〔2025〕12号第10条明确延长≥1年视为"连续订立" |
| 公司法第16条 | 公司为股东提供担保仅需董事会决议即可 | 司法解释明确需股东会决议 |
| 合同法第52条 | 违反管理性强制性规定的合同无效 | 司法解释区分管理性与效力性强制性规定 |

### 1.3 文件批量模式 — 引用提取

对上传的文件，使用 `scripts/parse_citations.py` 提取法律引用：

```bash
python scripts/parse_citations.py <文件路径> --output citations.json
```

脚本支持 8 种引用模式的正则提取：
1. 依据/根据《XX法》第X条/款/项
2. 《XX法》第X条第X款第X项
3. 《XX法》第X款
4. 《XX法》第X项
5. 《XX法》规定/相关规定
6. 《XX法》第X条至第X条（范围引用）
7. XX法第X条（无书名号）
8. 《XX法》法释〔年份〕X号（司法解释号引用）

脚本自动按法律名称分组去重，输出标准化 JSON。

> 若脚本不可用或用户仅粘贴文本，则手动扫描以下模式：
> - "依据/根据《XX法》第X条/款/项"
> - "依据/根据《XX法》规定"
> - "《XX法》第X条"
> - 按法律名称分组，避免重复搜索

```
// 并行搜索组合2：具体条款
web_search("{法律全称} 第{X}条 全文 官方")

```

```

web_search("{法律全称} 修订 废止 最新版本")
web_search("{法律全称} 现行有效")
```

### 2.4 搜索结果获取

从搜索结果中识别官方来源页面 URL，使用 `web_fetch` 获取法律全文。

若官方网站无法访问或未找到，按以下顺序降级：
1. 第二优先官方来源
2. 辅助参考来源（北大法宝、法信）— 仅作交叉验证
3. 通用搜索结果 — 标记为低置信度

> **检查点 CP2**：必须确认获取的是**最新版本**的官方原文。参考 `references/frequently-revised-laws.md` 中的修法时间线，确认版本对应。

## 步骤四：执行核查与比对

### 4.1 内容真实性比对

获取官方原文后，按 `references/comparison-rules.md` 中的规则进行系统性比对。

**比对维度与重点**：

| 比对维度 | 重点核查项 | 严重程度 |
|---------|-----------|---------|
| 逻辑连词 | "且"↔"或"、"并"↔"或" | 🔴 严重 |
| 模态词 | "应当"↔"可以"、"有权"↔"应当" | 🔴 严重 |
| 否定词 | "不得"↔"不应"、"未"↔"不" | 🟡 实质 |
| 数字期限 | 日数、年限、比例、金额、人数 | 🔴 严重 |
| 主体名称 | "法人"↔"法定代表人"、"公司"↔"企业" | 🔴/🟡 |
| 但书/除外 | 遗漏但书改变了适用范围 | 🔴 严重 |
| 条款对应 | 条款编号与内容是否对应 | 🔴 严重 |
| 款项层级 | 引用精度（条/款/项） | 🟡 实质 |
| 标点空格 | 逗号/句号/分号互换 | 🟢 轻微 |

**比对结果分级**：

| 级别 | 代号 | 含义 | 颜色标记 |
|------|------|------|---------|
| 完全一致 | ID | 逐字一致 | 🟢 |
| 轻微差异 | MN | 标点/格式差异，意思不变 | 🟡 |
| 实质差异 | SD | 关键词/数字/主体差异 | 🟠 |
| 严重差异 | CD | 逻辑词/模态词互换，意思相反 | 🔴 |
| 无法核实 | UV | 未找到官方原文 | ⚪ |

> 详见 `references/comparison-rules.md` 中的完整敏感词表和分级判定流程。

### 4.2 法律有效性核查

确认以下五项（缺一不可）：

- [ ] 该法律是否仍然有效（未被废止）
- [ ] 是否有更新版本（用户引用的可能是旧版）
- [ ] 用户引用的具体条款在最新版中是否变动、删除或移位
- [ ] 施行日期（确认已生效）
- [ ] 若已废止：找到替代的新法律及对应条款

**废止法律→替代法律映射**（详见 `references/frequently-revised-laws.md`）：
- 《婚姻法》《继承法》《合同法》《物权法》《侵权责任法》《担保法》《收养法》《民法通则》《民法总则》→ 《民法典》各编
- 《执业医师法》→ 《医师法》
- 《外资企业法》《中外合资经营企业法》《中外合作经营企业法》→ 《外商投资法》

### 4.3 置信度评估（文本比对维度）

根据文本比对和有效性核查过程中的来源可靠性和完整性，评估结果置信度：

| 等级 | 代号 | 条件 |
|------|------|------|
| **高** | H | ≥2个官方来源 + 来源一致 + 确认现行有效 + 确认最新版本 + 逐字比对完成 + 但书已检查 |
| **中** | M | 1个官方+1个辅助来源一致 / 2个辅助来源一致 / 无法确认最新版本 |
| **低** | L | 仅1个非官方来源 / 来源不一致 / 法律名称不确定 |

> 详见 `references/verification-checklist.md` 中的完整评估标准和因子表。

## 步骤五：司法解释与适用性审查（v1.2.0 独立步骤）

> ⚠️ **本步骤是v1.2.0的核心变更，从原步骤4.3升级为独立步骤。**
>
> **核心原则：法条文本正确 ≠ 法律适用正确。即便文本完全一致且法律现行有效，司法解释或配套规定可能已改变/明确了该条款的理解方式。**
>
> **本步骤不可跳过。无论步骤四的文本比对结果如何（即使是ID完全一致），都必须执行本步骤。**

### 5.1 关键词提取（v1.2.0 新增）

从用户引用的法条文本中，提取可能被司法解释重新定义的**关键法律概念**。

**提取规则**：
1. 识别法条中的**核心法律术语**（如"订立""解除""赔偿""连带责任"等）
2. 识别**条件性表述**（如"连续""累计""达到"等可能涉及计算标准的词）
3. 识别**范围性表述**（如"包括""但不限于""除外"等可能被扩展或限缩的词）
4. 结合**用户适用意图**，提取用户理解与法条字面含义的**分歧点关键词**

**提取示例**：

| 法条 | 提取的关键词 | 分歧点关键词（结合用户意图） |
|------|------------|-------------------------|
| 劳动合同法第14条第2款第3项 | "订立""连续""二次""固定期限" | **"订立"**（用户认为仅指重新签订，不含延长） |
| 公司法第16条 | "担保""股东""实际控制人""决议" | **"决议"**（用户认为董事会决议即可） |

### 5.2 双轨司法解释搜索（v1.2.0 核心策略）

**必须同时执行两条搜索轨道**（与步骤二的搜索并行）：

#### 轨道一：法条级搜索（原策略）

搜索该法律的整体司法解释体系：

```
// 必须执行
web_search("{法律名称} 司法解释 第{X}条")
web_search("最高人民法院 {法律名称} 适用法律 若干问题的解释")
web_search("{法律名称} 司法解释 最新 {当前年份}")
```

#### 轨道二：关键词级搜索（v1.2.0 新增）⭐

**使用步骤5.1提取的关键词，搜索是否有司法解释对该关键词作出了不同解释**：

```
// 必须执行 — 使用分歧点关键词
web_search("{法律名称} {分歧点关键词} 司法解释 延长 变更")
web_search("最高人民法院 {分歧点关键词} 含义 解释 {法律名称}")

// 示例：劳动合同法第14条的搜索
// web_search("劳动合同法 订立 延长 司法解释")
// web_search("最高人民法院 连续订立 含义 解释 劳动合同法")
```

**关键词级搜索的原理**：司法解释往往不是以"第X条"为标题来命名的，而是以**法律概念**为对象作出解释。搜索"劳动合同法第14条司法解释"可能返回笼统结果，但搜索"劳动合同法 订立 延长 司法解释"能精准命中法释〔2025〕12号第10条。

#### 轨道三：配套规定搜索（视情况追加）

发现司法解释线索时，或法条有明确配套行政法规时追加：

```
web_search("{法律名称} 第{X}条 实施条例 配套规定")
web_search("{法律名称} 实施条例 第{X}条")
```

#### 轨道四：指导性案例搜索（视情况追加）

司法解释搜索未发现关键信息时追加：

```
web_search("最高人民法院 指导性案例 {法律名称} 第{X}条")
web_search("最高法 指导案例 {分歧点关键词}")
```

> 搜索策略详见 `references/judicial-interpretation-search.md`。

### 5.3 解释性差异判定

当发现司法解释或配套规定与用户的理解不一致时，判定为"解释性差异"（IC — Interpretive Conflict）。

| 级别 | 代号 | 含义 | 典型场景 |
|------|------|------|---------|
| 严重解释性差异 | IC-CD | 司法解释明确否定了用户的理解 | 司法解释将"订立"明确包含"延长" |
| 实质解释性差异 | IC-SD | 司法解释改变了适用条件 | 司法解释新增了适用例外情形 |
| 轻微解释性差异 | IC-MN | 司法解释补充了未明确事项 | 司法解释明确了计算标准 |
| 提示性信息 | IC-Info | 配套规定细化了适用标准 | 实施条例细化了申请时限 |

**解释性差异（IC）与文本差异（ID/MN/SD/CD/UV）是并行独立维度。**

一个核验结果可能同时具有：文本差异=ID（完全一致）+ 解释性差异=IC-CD（司法解释否定了理解）。

**核心原则：文本完全一致 ≠ 法律适用正确**

### 5.4 判定流程

```
步骤四文本比对完成
│
├─ 用户提供了适用意图？
│  ├─ 是 → 提取分歧点关键词 → 执行双轨搜索
│  └─ 否 → 仅提取法条关键词 → 执行双轨搜索（分歧点关键词为空）
│
├─ 轨道一搜索（法条级）→ 找到司法解释
│  ├─ 与用户理解一致 → 无解释性差异
│  ├─ 司法解释否定了用户理解 → IC-CD 🔴
│  ├─ 司法解释改变了适用条件 → IC-SD 🟠
│  ├─ 司法解释补充了未明确事项 → IC-MN 🟡
│  └─ 司法解释细化了适用标准 → IC-Info 🟢
│
├─ 轨道二搜索（关键词级）→ 找到司法解释
│  └─ 按上述标准判定
│
├─ 轨道一+二均未找到 → 注明"已执行法条级+关键词级搜索，未发现相关司法解释"
│
└─ 轨道三/四搜索 → 同上判定
```

### 5.5 置信度评估（解释性差异维度）

| 等级 | 代号 | 条件 |
|------|------|------|
| **高** | H | 已检索司法解释（法条级+关键词级）+ 已检索配套规定 + 已结合用户意图判定 |
| **中** | M | 仅执行法条级搜索未执行关键词级 / 用户未提供适用意图 / 搜索结果不明确 |
| **低** | L | 未检索司法解释 / 司法解释搜索被跳过 |

> **检查点 CP3**：司法解释与配套规定核查必须完成。若用户未提供适用意图，必须主动询问后再完成本步骤。详见 `references/judicial-interpretation-search.md`。

## 步骤六：输出核验报告

### 6.1 核验自检

输出报告前，**必须**对照 `references/verification-checklist.md` 完成以下 12 项检查：

| 序号 | 检查项 | 常见遗漏 |
|------|--------|---------|
| C1 | 法律全称确认 | 用简称代替全称 |
| C2 | 官方原文获取 | 仅从非官方来源获取 |
| C3 | 来源数量≥2 | 仅搜了1次就下结论 |
| C4 | 废止状态确认 | 忘记查废止状态 |
| C5 | 最新版本确认 | 获取了旧版而非最新版 |
| C6 | 条款编号验证 | 引用了旧版条款号 |
| C7 | 内容逐字比对 | 仅做了大意比对 |
| C8 | 例外/但书检查 | 遗漏但书改变适用范围 |
| C9 | 司法解释法条级搜索 | 未搜索司法解释导致法律适用错误 |
| C10 | 司法解释关键词级搜索 | 法条级搜索未找到但关键词级可找到 |
| C11 | 配套规定核查 | 未注意到配套规定改变适用条件 |
| C12 | 用户适用意图确认 | 未了解用户如何理解/适用该条款 |

**通过标准**：
- ✅ 12项全部确认 → 可输出确定性结论
- ⚠️ 10-11项确认 → 输出结论但标注未确认项
- ❌ 9项以下 → 不可输出核验结论，仅列出已获取信息

## 特殊情况处理

### 无法找到官方原文时

> ⚪ **未能核实**：在官方数据库中未能找到《XXX》第X条的官方原文。可能原因：(1) 该法条不存在于现行法律体系；(2) 属于内部规定或地方性规范性文件；(3) 法律名称有误。建议通过 [flk.npc.gov.cn](https://flk.npc.gov.cn) 进行人工查询，或咨询专业律师。

置信度标记为 L，不可输出确定性结论。

### 版本敏感提示

许多法律经历多次修订（如《著作权法》《公司法》《民事诉讼法》等），用户可能引用旧版条文。即便内容本身曾经存在，也须明确指出是否为**当前有效版本**。

参考 `references/frequently-revised-laws.md` 中的修法时间线，对🔴极高敏感度和🟡中等敏感度法律自动触发额外版本检查。

### 条款编号变动

修订后条款编号可能移位（如《公司法》2023年修订后多处条款编号变动），须明确告知用户当前版本中的正确条款位置。

参考 `references/frequently-revised-laws.md` 中的条款编号变动对照表。

### 多版本条文共存

部分法律存在新旧版本过渡期（如《公司法》2023年修订版2024年7月1日施行，旧版在此日期前仍有效）。须根据用户引用的具体时间场景判断适用版本。

### 用户未提供适用意图

若用户仅提供法条文本而未说明适用意图：
1. **必须主动询问**：在步骤五开始前，询问用户"请问您基于该条款的法律论点或适用方式是什么？"
2. **若用户拒绝提供**：仍执行双轨搜索，但仅能报告"发现相关司法解释"而无法判定是否存在解释性差异（因为不知道用户的理解），置信度不超过M
3. **批量模式**：从文件的法律论述中提取适用意图；若文件仅列法条无论述，标记为"未提供"
