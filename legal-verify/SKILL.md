# 法条校验技能（Legal-Verify）

## 技能定位

本技能对文本中援引的法律条文执行**对抗性验证**（adversarial verification）——
主动寻找引用错误，而非被动接受。输出结果为供律师审阅的校验报告草稿，
不构成法律意见，不替代执业律师的专业判断。

> **引用可信度标注规则**（沿用 Claude for Legal 体系）
> - 通过研究连接器（MCP）实时核实的引用 → 标注来源 `[来源: XXX]`
> - 基于模型知识推断的引用 → 标注 `[待核实]`
> - 无研究工具接入时 → 报告头部注明"法条来源未经实时数据库核实"

---

## 激活场景

以下任一情形自动激活本技能：

- 文本中出现法律编号引用（如"《合同法》第52条"、"《民法典》第577条"）
- 用户要求核实某法律文件的法条依据
- 合同、诉状、法律意见书、合规报告需要法条校验
- 用户询问某条文是否"现行有效"、"是否已修订"
- 用户粘贴法律条文、上传含有法律引用的文件

---

## 校验深度选项

| 深度 | 适用场景 | 覆盖范围 |
|-----|---------|---------|
| **快速** | 合同初审、快速浏览 | 步骤二（时效性）+ 步骤三（明显错误） |
| **标准**（默认） | 正式文件审查 | 全流程七步 |
| **深度** | 诉状、法律意见书、上市文件 | 全流程 + 引用链路追踪 + 司法解释四轨搜索 |

深度由用户指定或根据文件类型自动推断：合同/诉状/法律意见书默认"深度"，普通文件默认"标准"。

## 步骤一：提取与结构化引用清单

### 1.1 扫描与提取

扫描输入文本，提取所有法律引用，结构化为：

```
引用ID | 法律名称 | 条款编号 | 援引声称的含义 | 出现位置
```

对每个引用标注初始可信度：
- `高风险` — 法律名称与版本存在明显歧义（如"合同法"在《民法典》施行后）
- `中风险` — 条款编号与通常认知不符
- `低风险` — 格式标准、引用语义合理

### 1.2 用户适用意图

> ⚠️ **这是解决司法解释检测盲区的关键输入。**

法条校验存在两种截然不同的错误类型：
1. **文本错误**：法条文本与官方原文不一致 → 不需要用户意图即可检测
2. **适用错误**：法条文本完全正确，但司法解释已改变/明确了该条款的理解方式 → **必须知道用户怎么理解/适用该条款**，才能判断是否存在解释性差异

**如何获取用户适用意图？**

- **用户主动提供**：如"我认为订立不包含延长""依据该条主张XX权利"
- **从上下文推断**：用户提到"根据第X条，所以YY"→ "YY"就是适用意图
- **若用户未提供**：**必须主动询问**——"请问您基于该条款的法律论点或适用方式是什么？这有助于检测是否存在司法解释改变了该条款的理解方式。"
- **批量模式**：若文件中包含法律论述，从论述中提取适用意图；若仅列出法条引用无论述，标记为"未提供"

**用户适用意图示例**：

| 法条 | 用户的适用意图 | 可能的解释性差异 |
|------|--------------|----------------|
| 劳动合同法第14条第2款第3项 | "订立"指重新签订合同，延长协议不计入"连续订立"次数 | 法释〔2025〕12号第10条明确延长≥1年视为"连续订立" |
| 公司法第16条 | 公司为股东提供担保仅需董事会决议即可 | 司法解释明确需股东会决议 |
| 合同法第52条 | 违反管理性强制性规定的合同无效 | 司法解释区分管理性与效力性强制性规定 |

> **检查点 CP1**：法律名称必须使用全称（含"中华人民共和国"前缀）。若用户使用简称（如"劳动法"），须确认是指《劳动法》还是《劳动合同法》，避免混淆。

### 1.3 文件批量模式 — 引用提取

对上传的文件，使用 `scripts/parse_citations.py` 提取法律引用：

```bash
python scripts/parse_citations.py <文件路径> --output citations.json
```

脚本支持 8 种引用模式的正则提取：
1. 依据/根据《XX法》第X条/款/项
2. 《XX法》第X条第X款第X项
3. 《XX法》第X款
4. 《XX法》第X项
5. 《XX法》规定/相关规定
6. 《XX法》第X条至第X条（范围引用）
7. XX法第X条（无书名号）
8. 《XX法》法释〔年份〕X号（司法解释号引用）

脚本自动按法律名称分组去重，输出标准化 JSON。

> 若脚本不可用或用户仅粘贴文本，则手动扫描以下模式：
> - "依据/根据《XX法》第X条/款/项"
> - "依据/根据《XX法》规定"
> - "《XX法》第X条"
> - 按法律名称分组，避免重复搜索

## 步骤二：时效性验证（Temporal Confidence Check）

> ⚠️ 这是法条校验中**最易出错**的环节，必须优先处理。

### 2.1 时效链追踪

对每条引用执行时效链追踪：

```
[原始法律] → [历次修订] → [当前有效版本] → [是否已被整合入新法]
```

### 2.2 关键时效节点（中国法律体系）

| 时效节点 | 事件 | 影响范围 |
|---------|------|---------|
| 2021-01-01 | 《民法典》施行 | 《合同法》《物权法》《侵权责任法》《担保法》等9部法律同步废止 |
| 2023-07-01 | 《公司法》修订版施行 | 注册资本缴足期限等重大变化，条款从218条→266条 |
| 2024-01-01 | 《行政复议法》修订版施行 | 条款从43条→86条 |
| 监管类法规 | 变动频繁 | 务必标注 `[待核实当前版本]` |

### 2.3 版本敏感法条自动预警

当核验的法律在 `references/frequently-revised-laws.md` 中被标记为🔴极高敏感度或🟡中等敏感度时，自动触发额外版本检查：

**🔴 极高敏感度法律**（条款重排，旧号几乎全部失效）：
- 《公司法》（2023年修订，2024.7.1施行）
- 《行政复议法》（2023年修订，2024.1.1施行）
- 民法典相关旧法引用（婚姻法、继承法、合同法、物权法等→民法典）

**🟡 中等敏感度法律**（部分条款变动）：
- 《劳动合同法》（2012年修正）
- 《行政处罚法》（2021年修订）
- 《反垄断法》（2022年修正）
- 《安全生产法》（2021年修正）
- 《专利法》（2020年修正）
- 《著作权法》（2020年修正）

```
✓ 现行有效  /  ✗ 已废止（废止日期：XXXX）  /  ⚠ 已修订（引用版本为旧版）  /  ❓ [待核实]
```

## 步骤三：原文比对

将援引声称的含义与条文原文进行语义比对，识别以下错误类型：

| 错误类型 | 说明 | 严重程度 |
|---------|------|---------|
| **原文曲解** | 条文含义被扩大/缩小/反转 | 🔴 严重 |
| **条款错位** | 引用内容属于相邻条款而非所引条款 | 🔴 严重 |
| **版本错误** | 引用旧版条文，现行版本已修订 | 🟠 重要 |
| **格式不规范** | 编号格式不符合规范，但内容无误 | 🟡 轻微 |
| **引用冗余** | 引用条款与论证无实质关联 | 🟡 轻微 |
| **管辖区错配** | 引用法律与适用管辖区不符 | 🔴 严重 |

### 3.1 内容真实性比对

获取官方原文后，按 `references/comparison-rules.md` 中的规则进行系统性比对。

**比对维度与重点**：

| 比对维度 | 重点核查项 | 严重程度 |
|---------|-----------|---------|
| 逻辑连词 | "且"↔"或"、"并"↔"或" | 🔴 严重 |
| 模态词 | "应当"↔"可以"、"有权"↔"应当" | 🔴 严重 |
| 否定词 | "不得"↔"不应"、"未"↔"不" | 🟡 实质 |
| 数字期限 | 日数、年限、比例、金额、人数 | 🔴 严重 |
| 主体名称 | "法人"↔"法定代表人"、"公司"↔"企业" | 🔴/🟡 |
| 但书/除外 | 遗漏但书改变了适用范围 | 🔴 严重 |
| 条款对应 | 条款编号与内容是否对应 | 🔴 严重 |
| 款项层级 | 引用精度（条/款/项） | 🟡 实质 |
| 标点空格 | 逗号/句号/分号互换 | 🟢 轻微 |

### 3.2 比对结果分级

| 级别 | 代号 | 含义 | 颜色标记 |
|------|------|------|---------|
| 完全一致 | ID | 逐字一致 | 🟢 |
| 轻微差异 | MN | 标点/格式差异，意思不变 | 🟡 |
| 实质差异 | SD | 关键词/数字/主体差异 | 🟠 |
| 严重差异 | CD | 逻辑词/模态词互换，意思相反 | 🔴 |
| 无法核实 | UV | 未找到官方原文 | ⚪ |

> 详见 `references/comparison-rules.md` 中的完整敏感词表和分级判定流程。

### 3.3 法律有效性核查

确认以下五项（缺一不可）：

- [ ] 该法律是否仍然有效（未被废止）
- [ ] 是否有更新版本（用户引用的可能是旧版）
- [ ] 用户引用的具体条款在最新版中是否变动、删除或移位
- [ ] 施行日期（确认已生效）
- [ ] 若已废止：找到替代的新法律及对应条款

### 3.4 置信度评估（文本比对维度）

| 等级 | 代号 | 条件 |
|------|------|------|
| **高** | H | ≥2个官方来源 + 来源一致 + 确认现行有效 + 确认最新版本 + 逐字比对完成 + 但书已检查 |
| **中** | M | 1个官方+1个辅助来源一致 / 2个辅助来源一致 / 无法确认最新版本 |
| **低** | L | 仅1个非官方来源 / 来源不一致 / 法律名称不确定 |

## 步骤四：司法解释与适用性审查

> **核心原则：法条文本正确 ≠ 法律适用正确。即便文本完全一致且法律现行有效，司法解释或配套规定可能已改变/明确了该条款的理解方式。**
>
> **本步骤不可跳过。无论步骤三的文本比对结果如何（即使是ID完全一致），都必须执行本步骤。**

### 4.1 关键词提取

从用户引用的法条文本中，提取可能被司法解释重新定义的**关键法律概念**。

**提取规则**：
1. 识别法条中的**核心法律术语**（如"订立""解除""赔偿""连带责任"等）
2. 识别**条件性表述**（如"连续""累计""达到"等可能涉及计算标准的词）
3. 识别**范围性表述**（如"包括""但不限于""除外"等可能被扩展或限缩的词）
4. 结合**用户适用意图**，提取用户理解与法条字面含义的**分歧点关键词**

**提取示例**：

| 法条 | 提取的关键词 | 分歧点关键词（结合用户意图） |
|------|------------|-------------------------|
| 劳动合同法第14条第2款第3项 | "订立""连续""二次""固定期限" | **"订立"**（用户认为仅指重新签订，不含延长） |
| 公司法第16条 | "担保""股东""实际控制人""决议" | **"决议"**（用户认为董事会决议即可） |

### 4.2 多轨司法解释搜索

**必须同时执行两条核心搜索轨道**（与步骤二的搜索并行）：

#### 轨道一：法条级搜索

搜索该法律的整体司法解释体系：

```
// 必须执行
web_search("{法律名称} 司法解释 第{X}条")
web_search("最高人民法院 {法律名称} 适用法律 若干问题的解释")
web_search("{法律名称} 司法解释 最新 {当前年份}")
```

#### 轨道二：关键词级搜索 ⭐

**使用步骤4.1提取的关键词，搜索是否有司法解释对该关键词作出了不同解释**：

```
// 必须执行 — 使用分歧点关键词
web_search("{法律名称} {分歧点关键词} 司法解释 延长 变更")
web_search("最高人民法院 {分歧点关键词} 含义 解释 {法律名称}")
```

**关键词级搜索的原理**：司法解释往往不是以"第X条"为标题来命名的，而是以**法律概念**为对象作出解释。搜索"劳动合同法第14条司法解释"可能返回笼统结果，但搜索"劳动合同法 订立 延长 司法解释"能精准命中法释〔2025〕12号第10条。

#### 轨道三：配套规定搜索（深度模式必须执行）

发现司法解释线索时，或法条有明确配套行政法规时执行：

```
web_search("{法律名称} 第{X}条 实施条例 配套规定")
web_search("{法律名称} 实施条例 第{X}条")
```

#### 轨道四：指导性案例搜索（深度模式必须执行）

```
web_search("最高人民法院 指导性案例 {法律名称} 第{X}条")
web_search("最高法 指导案例 {分歧点关键词}")
```

> 搜索策略详见 `references/judicial-interpretation-search.md`。

### 4.3 解释性差异判定

当发现司法解释或配套规定与用户的理解不一致时，判定为"解释性差异"（IC — Interpretive Conflict）。

| 级别 | 代号 | 含义 | 典型场景 |
|------|------|------|---------|
| 严重解释性差异 | IC-CD | 司法解释明确否定了用户的理解 | 司法解释将"订立"明确包含"延长" |
| 实质解释性差异 | IC-SD | 司法解释改变了适用条件 | 司法解释新增了适用例外情形 |
| 轻微解释性差异 | IC-MN | 司法解释补充了未明确事项 | 司法解释明确了计算标准 |
| 提示性信息 | IC-Info | 配套规定细化了适用标准 | 实施条例细化了申请时限 |
| 发现但无法判定 | IC-Pending | 发现司法解释但用户未提供适用意图 | — |

**解释性差异（IC）与文本差异（ID/MN/SD/CD/UV）是并行独立维度。**

一个校验结果可能同时具有：文本差异=ID（完全一致）+ 解释性差异=IC-CD（司法解释否定了理解）。

### 4.4 判定流程

```
步骤三文本比对完成
│
├─ 用户提供了适用意图？
│  ├─ 是 → 提取分歧点关键词 → 执行多轨搜索
│  └─ 否 → 主动询问 → 若仍未提供 → 执行多轨搜索但标记IC-Pending
│
├─ 轨道一搜索（法条级）→ 找到司法解释
│  ├─ 与用户理解一致 → 无解释性差异
│  ├─ 司法解释否定了用户理解 → IC-CD 🔴
│  ├─ 司法解释改变了适用条件 → IC-SD 🟠
│  ├─ 司法解释补充了未明确事项 → IC-MN 🟡
│  └─ 司法解释细化了适用标准 → IC-Info 🟢
│
├─ 轨道二搜索（关键词级）→ 找到司法解释
│  └─ 按上述标准判定（关键词级可能命中法条级未命中的结果）
│
├─ 轨道一+二均未找到 → 注明"已执行法条级+关键词级搜索，未发现相关司法解释"
│
└─ 轨道三/四搜索 → 同上判定
```

### 4.5 置信度评估（解释性差异维度）

| 等级 | 代号 | 条件 |
|------|------|------|
| **高** | H | 已检索司法解释（法条级+关键词级）+ 已检索配套规定 + 已结合用户意图判定 |
| **中** | M | 仅执行法条级搜索未执行关键词级 / 用户未提供适用意图 / 搜索结果不明确 |
| **低** | L | 未检索司法解释 / 司法解释搜索被跳过 |

> **检查点 CP3**：司法解释与配套规定核查必须完成。若用户未提供适用意图，必须主动询问后再完成本步骤。详见 `references/judicial-interpretation-search.md`。

## 步骤五：管辖区适配检查

明确文件适用管辖区，检查：

- 是否援引了错误管辖区的法律（如在香港适用的合同引用大陆法）
- 地方性法规（省/市级）与国家层面法律是否存在冲突
- 涉外合同是否考虑了准据法选择

**管辖区假设未明确时：在报告顶部显式标注推断依据，不得静默假设。**

### 5.1 管辖区检查项

| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 法域错配 | 在大陆适用的文件引用了港澳台或境外法律 | 🔴 严重 |
| 地方性法规冲突 | 地方性法规与上位法不一致 | 🟠 重要 |
| 准据法缺失 | 涉外合同未约定准据法 | 🟡 轻微 |
| 过度引用 | 不需要引用境外法律的情形引用了境外法律 | 🟡 轻微 |

| 国家法律法规数据库（flk.npc.gov.cn） | 法律时效性核查 |
| CourtListener / Westlaw | 涉外/港澳台法律核实 |

**未接入任何研究工具时**：报告顶部必须标注"法条来源未经实时数据库核实，建议逐条通过权威数据库复核"，所有引用一律标注 `[待核实]`。

## 边界与护栏

- **不提供法律意见**：校验结论仅针对引用准确性，不判断法律策略的妥当性
- **不直接修改源文件**：仅提供修正建议，实际修改由律师决定并执行
- **司法解释谨慎处理**：最高法解释、部门规章、地方规范性文件变动频繁，凡基于模型知识核实的此类引用，一律标注 `[待核实]`
- **争议法律领域**：如引用涉及尚无定论的法律解释，明确标注"存在解释争议"，不作倾向性结论
- **保密提示**：如文档包含当事人信息，提醒用户在使用外部数据库核实时注意保密义务

## 特殊情况处理

### 无法找到官方原文时

> ⚪ **未能核实**：在官方数据库中未能找到《XXX》第X条的官方原文。可能原因：(1) 该法条不存在于现行法律体系；(2) 属于内部规定或地方性规范性文件；(3) 法律名称有误。建议通过 [flk.npc.gov.cn](https://flk.npc.gov.cn) 进行人工查询，或咨询专业律师。

置信度标记为 L，不可输出确定性结论。

### 版本敏感提示

许多法律经历多次修订（如《著作权法》《公司法》《民事诉讼法》等），用户可能引用旧版条文。即便内容本身曾经存在，也须明确指出是否为**当前有效版本**。

参考 `references/frequently-revised-laws.md` 中的修法时间线，对🔴极高敏感度和🟡中等敏感度法律自动触发额外版本检查。

### 条款编号变动

修订后条款编号可能移位（如《公司法》2023年修订后多处条款编号变动），须明确告知用户当前版本中的正确条款位置。

参考 `references/frequently-revised-laws.md` 中的条款编号变动对照表。

### 多版本条文共存

部分法律存在新旧版本过渡期（如《公司法》2023年修订版2024年7月1日施行，旧版在此日期前仍有效）。须根据用户引用的具体时间场景判断适用版本。

### 用户未提供适用意图

若用户仅提供法条文本而未说明适用意图：
1. **必须主动询问**：在步骤四开始前，询问用户"请问您基于该条款的法律论点或适用方式是什么？"
2. **若用户拒绝提供**：仍执行多轨搜索，但仅能报告"发现相关司法解释"而无法判定是否存在解释性差异（因为不知道用户的理解），标注IC-Pending，置信度不超过M
3. **批量模式**：从文件的法律论述中提取适用意图；若文件仅列法条无论述，标记为"未提供"

### 管辖区未明确时

1. 从文件上下文推断（签署地、履行地、当事人住所地等）
2. 在报告顶部显式标注："管辖区推断为XXX，依据：XXX"
3. 不得静默假设管辖区
