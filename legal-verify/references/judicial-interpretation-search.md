# 司法解释与配套规定搜索指南（v1.2.0）

> ⚠️ 本文件解决的核心问题：法条文本本身正确且现行有效，但司法解释或配套规定已改变或明确了该条款的理解方式，引用者因未注意到而得出错误结论。
>
> **v1.2.0 核心变更**：新增"关键词级搜索"策略和"用户适用意图驱动的搜索"策略，解决法条级搜索无法精准命中司法解释的问题。

---

## 一、问题场景说明

### 1.1 典型案例

律师引用《劳动合同法》第14条第2款第3项"连续订立二次固定期限劳动合同"条款，法条文本完全正确、法律现行有效，但未注意到《最高人民法院关于审理劳动争议案件适用法律问题的解释（二）》（法释〔2025〕12号）第10条已明确：延长劳动合同期限累计1年以上视为"连续订立二次"。律师仅从字面理解"订立"vs"变更"，导致法律适用错误。

### 1.2 v1.1.0的搜索为什么没找到？

| 搜索策略 | 查询示例 | 结果 |
|---------|---------|------|
| 法条级搜索 | "劳动合同法 司法解释 第14条" | 可能返回劳动争议司法解释（一）~（五）的笼统结果 |
| 法条级搜索 | "最高人民法院 劳动合同法 适用法律 若干问题的解释" | 同上，无法精准定位到"订立"这个词的解释 |
| **关键词级搜索** ⭐ | "劳动合同法 订立 延长 司法解释" | **精准命中法释〔2025〕12号第10条** |
| **关键词级搜索** ⭐ | "最高人民法院 连续订立 含义 解释" | **同上** |

**结论**：法条级搜索太泛，搜索"第14条司法解释"返回的是整部司法解释，而司法解释往往以**法律概念**为标题，不是以条款号为标题。必须用关键词精准搜索。

### 1.3 问题本质

| 维度 | 当前技能已覆盖 | v1.2.0新增 |
|------|--------------|-----------|
| 法条文本是否与官方一致 | ✅ 内容真实性比对 | — |
| 法条是否现行有效 | ✅ 法律有效性核查 | — |
| 有无司法解释改变/明确了该条款的理解 | ✅ 法条级搜索 | ✅ **关键词级搜索** |
| 用户的法律适用是否正确 | ❌ 不知道用户怎么理解 | ✅ **用户适用意图驱动** |
| 有无配套规定补充了适用条件 | ✅ 配套规定搜索 | — |

## 二、双轨搜索策略

### 2.1 总览

| 搜索轨道 | 策略 | 必须执行 | 搜索目标 |
|---------|------|---------|---------|
| 轨道一 | 法条级搜索 | ✅ | 找到该法律的整体司法解释体系 |
| 轨道二 | 关键词级搜索 | ✅ | 精准找到对该法律概念作出解释的司法解释 |
| 轨道三 | 配套规定搜索 | ⚠️ 视情况 | 找到实施条例等配套规定 |
| 轨道四 | 指导性案例搜索 | ⚠️ 视情况 | 找到改变理解的指导性案例 |

### 2.2 轨道一：法条级搜索

#### 搜索引擎查询模板

```
// 组合1：按法律名称搜索司法解释
web_search("site:court.gov.cn 最高人民法院 {法律名称} 解释")
web_search("最高人民法院 关于 {法律名称} 适用法律 若干问题的解释")

// 组合2：按具体条款搜索司法解释
web_search("{法律全称} 第{X}条 司法解释")
web_search("最高人民法院 {法律名称} 第{X}条 解释 规定")

// 组合3：按文号搜索
web_search("法释〔{年份}〕{X}号 {法律名称}")
web_search("法释 {年份} {X}号 {条款关键词}")
```

#### 重要：搜索时机
**每次核验都必须执行法条级搜索**，即使用户只要求核实法条文本。司法解释可能：
- 扩展法条的适用范围
- 限缩法条的适用条件
- 改变法条中关键词的法律含义
- 明确法条中模糊条款的具体适用标准

### 2.3 轨道二：关键词级搜索 ⭐（v1.2.0 核心）

#### 搜索原理

司法解释不是以"第X条"为标题来命名的，而是以**法律概念**为对象作出解释。例如：
- 法释〔2025〕12号第10条的标题是关于"延长劳动合同期限"的规定，不是关于"第14条"的规定
- 因此搜索"第14条司法解释"很难命中，但搜索"订立 延长 司法解释"可以精准命中

#### 关键词提取规则

在执行关键词级搜索前，必须先从法条文本和用户适用意图中提取关键词：

**从法条文本提取**：
1. **核心法律术语**：如"订立""解除""赔偿""连带责任""善意取得"等
2. **条件性表述**：如"连续""累计""达到""超过"等可能涉及计算标准的词
3. **范围性表述**：如"包括""但不限于""除外""视为"等可能被扩展或限缩的词
4. **时间性表述**：如"一年以上""六十日内"等可能被细化的期限

**从用户适用意图提取**（更关键）：
1. **分歧点关键词**：用户理解与法条字面含义不同的地方
   - 示例：用户认为"订立"仅指重新签订 → "订立"就是分歧点关键词
   - 示例：用户认为"担保"仅需董事会决议 → "担保决议"就是分歧点关键词
2. **反向关键词**：用户理解的反面
   - 示例：用户认为"订立"不含"延长" → 反向关键词是"延长""变更"
   - 示例：用户认为"担保"无需股东会决议 → 反向关键词是"股东会决议""股东大会"

#### 搜索引擎查询模板

```
// 组合1：分歧点关键词 + 司法解释
web_search("{法律名称} {分歧点关键词} 司法解释")
web_search("最高人民法院 {分歧点关键词} 含义 解释 {法律名称}")

// 组合2：分歧点关键词 + 反向关键词
web_search("{法律名称} {分歧点关键词} {反向关键词} 司法解释")
web_search("最高人民法院 {分歧点关键词} 是否包含 {反向关键词}")

// 组合3：法律概念解释
web_search("{法律名称} {核心法律术语} 如何理解 司法解释")
web_search("最高人民法院 {核心法律术语} 认定 {法律名称}")
```

#### 完整搜索示例

**案例：《劳动合同法》第14条第2款第3项**

用户适用意图："订立"仅指重新签订，"延长"不算"订立"

提取关键词：
- 核心法律术语：订立、连续、二次、固定期限
- 分歧点关键词：**订立**
- 反向关键词：**延长、变更、续签**

搜索查询：
```
// 法条级搜索（轨道一）
web_search("劳动合同法 司法解释 第14条")
web_search("最高人民法院 劳动合同法 适用法律 若干问题的解释")

// 关键词级搜索（轨道二）— 这些是能精准命中法释〔2025〕12号的查询
web_search("劳动合同法 订立 延长 司法解释")
web_search("最高人民法院 连续订立 含义 解释 劳动合同法")
web_search("劳动合同法 订立 是否包含 延长 变更")
web_search("最高人民法院 延长劳动合同期限 视为 连续订立")
```

## 五、解释性差异判断规则

### 5.1 何时判定存在"解释性差异"

当满足以下条件时，判定为"解释性差异"（IC — Interpretive Conflict）：

1. 法条文本与官方原文**完全一致**（ID级）
2. 法律**现行有效**
3. 但存在司法解释/配套规定对用户引用的条款作出了**明确解释**
4. 该解释与用户的**理解/适用方式**不一致（**需要知道用户的适用意图**）

### 5.2 判断的关键：用户适用意图

> ⚠️ **没有用户的适用意图，无法判断解释性差异的严重程度。**

| 情况 | 用户意图 | 司法解释内容 | 判定 |
|------|---------|------------|------|
| 情况1 | "订立"仅指重新签订 | 延长≥1年视为连续订立 | **IC-CD** 司法解释否定了用户理解 |
| 情况2 | 未提供适用意图 | 延长≥1年视为连续订立 | 仅报告"发现相关司法解释"，不判定IC级别 |
| 情况3 | "订立"包含续签 | 延长≥1年视为连续订立 | 无解释性差异（用户理解与司法解释一致） |

### 5.3 解释性差异的严重程度

| 情形 | 严重程度 | 标记 |
|------|---------|------|
| 司法解释明确否定了用户的理解 | 🔴 严重 | IC-CD |
| 司法解释改变了法条的适用条件 | 🟠 实质 | IC-SD |
| 司法解释补充了法条未明确的事项 | 🟡 轻微 | IC-MN |
| 配套规定细化了适用标准（未改变理解方向） | 🟢 提示 | IC-Info |

### 5.4 与现有差异分级的关系

"解释性差异"（IC）是与"文本差异"（ID/MN/SD/CD/UV）**并行**的独立维度。一个核验结果可能同时具有：

- 文本差异级别：ID（文本完全一致）
- 解释性差异级别：IC-CD（司法解释否定了用户的理解）

**即：文本完全一致 ≠ 法律适用正确**

## 六、搜索执行要求

### 6.1 必须执行的搜索

对每一条核验请求，除原有的文本比对和有效性核查搜索外，**还必须**执行以下搜索：

```
// 必须执行 — 轨道一：法条级搜索
web_search("{法律名称} 司法解释 第{X}条")
web_search("最高人民法院 {法律名称} 适用法律 若干问题的解释")

// 必须执行 — 轨道二：关键词级搜索（v1.2.0新增）
web_search("{法律名称} {分歧点关键词} 司法解释")
web_search("{法律名称} {分歧点关键词} {反向关键词} 司法解释")

// 视情况执行 — 轨道三：配套规定搜索（发现司法解释线索时）
web_search("{法律名称} 第{X}条 实施条例 配套规定")

// 视情况执行 — 轨道四：指导性案例搜索（轨道一+二均未发现关键信息时）
web_search("最高人民法院 指导性案例 {法律名称} 第{X}条")
```

### 6.2 搜索结果处理

1. 若搜索到相关司法解释，**必须**在核验报告中单列"司法解释与适用性审查"区块
2. 即使未发现解释性差异，也应在报告中注明"已执行法条级+关键词级搜索，未发现与用户引用条款直接相关的司法解释"
3. 若搜索结果不明确，标注为"需进一步确认"，置信度不高于M
4. **轨道一和轨道二的搜索结果应分别记录**，即使最终结论相同

### 6.3 新司法解释追踪

司法解释持续更新，核验时必须确认搜索的是**最新**司法解释：

```
web_search("{法律名称} 司法解释 最新 {当前年份}")
web_search("最高人民法院 {法律名称} 解释 {当前年份}")
```

### 6.4 搜索失败处理

若法条级搜索和关键词级搜索均未找到任何相关司法解释：
1. 尝试降低搜索精度（去掉site限定、使用更少关键词）
2. 尝试搜索"法律名称 + 争议 + 条款关键词"
3. 若仍无结果，注明"已执行法条级+关键词级搜索，均未发现相关司法解释。建议用户通过court.gov.cn人工查询"
